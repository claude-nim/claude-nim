# AGENTS.md

## Build & Dev Commands

```bash
bun install                  # install deps
bun run compile              # precompile (inline assets) → tsc → out/
bun run test                 # unit tests only (excludes integration/)
bun run test:integration     # integration tests (needs live NVIDIA NIM key)
bun run test:all             # all tests
bun run lint                 # eslint on src/ and tests/
bun run lint:fix             # auto-fix lint
bun run format               # prettier on src/ and tests/
bun run package:vsix         # compile + vsce package
```

**Critical build detail:** `precompile` runs `scripts/inline-assets.js` before `tsc`. This inlines `src/dashboard.html` and `src/dashboard-client.js` into `src/dashboard/dashboard-assets.ts`. If you edit dashboard assets, re-run `bun run compile` — the generated file is gitignored.

## Architecture

Two entrypoints:
- **VS Code extension** (`src/extension/index.ts` → `out/extension.js`): registered in `package.json` as `main`. Commands, status bar, SecretStorage for API key.
- **Standalone CLI** (`src/cli/index.ts` → `out/cli.js`): mock-hacks `vscode` module at runtime via `Module.prototype.require` override so `src/server/index.ts` works outside VS Code. Spawned by `bunx claude-nim`.

**Request flow:** Claude Code → `localhost:3456` (Anthropic Messages format) → `src/translator/index.ts` (Anthropic→OpenAI) → NVIDIA NIM API.

**Server modules** in `src/server/`: `index.ts` (lifecycle), `routes.ts` (request routing), `proxy-state.ts`, `model-router.ts`, `models-handler.ts`, `reasoning-stripper.ts`, `retry.ts`, `rate-limiter.ts`, `gateway-model-ids.ts`, `proxy-tool.ts`, `web-search-tool.ts`, `web-search.ts`, `config.ts`, `nim-settings.ts`, `logger.ts`.

**CLI modules** in `src/cli/`: `index.ts` (raw-mode input, prompts, lifecycle), `cli-menu.ts` (terminal UI: main menu, list menu, keyboard input).

**Dashboard** in `src/dashboard/`: `index.ts` (live metrics SSE, request ring buffer, `metrics.jsonl` persistence).

## Testing

- Jest with ts-jest, `tests/__mocks__/vscode.ts` mocks the `vscode` module via `moduleNameMapper` in `jest.config.js`.
- `bun run test` excludes `tests/integration.test.ts` (requires live API key).
- Path alias `~/` maps to `src/` in tests.

## Linting

ESLint with `typescript-eslint` + `prettier`. Config in `eslint.config.js`.
- `@typescript-eslint/no-require-imports` is a warning (not error) — `require()` is used intentionally in `cli/index.ts` and `constants.ts`.
- `src/dashboard/dashboard-assets.ts` is ignored by ESLint (auto-generated).

## Conventions

- All source files have a copyright header — preserve it on edit.
- API key stored in VS Code SecretStorage (extension) or `~/.claude-nim-key` (CLI, AES-256-GCM encrypted).
- Proxy binds to `127.0.0.1` only — never `0.0.0.0`.

## Engineering Rules (from `src/RULES.md`)

- **File limits:** Max 700 lines, preferred ≤300 lines. Split files that exceed this.
- **One feature per file, one responsibility per module.** No god classes or massive utility files.
- **Composition over inheritance.** Use traits/interfaces and dependency injection.
- **No dead code or placeholders.** Remove unused code, don't leave stubs.
- **Performance:** Minimize allocations, cache expensive operations, prefer streaming/incremental processing.
- **Tests required:** Unit, integration, and performance tests where applicable.
- **Deterministic:** Every component must be deterministic, measurable, and testable.

## Memory & Performance

- **SSE ring buffer** (`api/index.ts`): Line scanning uses raw `Uint8Array` newline detection — avoids string concatenation per stream chunk. Only one incomplete line is retained between reads (no growing buffer).
- **Prune context** (`translator/index.ts`): Uses lazy shallow-clone instead of `JSON.parse(JSON.stringify(...))` — O(trimmed) not O(all messages).
- **Tool call args** (`translator/tool-parser.ts`): Accumulated via `string[]` push + join instead of repeated string `+=` (avoids N-1 reallocations).
- **Metric IDs** (`constants.ts`): Monotonic counter + one timestamp per ID — no random string allocations per request.
- **All source files under 420 lines** — refactor if you exceed 700.

## Gotchas

- `out/` is the compiled output and is gitignored. Always `bun run compile` before `package:vsix`.
- The `.env` file contains a test API key — it's gitignored but present in the repo. Do not commit additional secrets.
- `src/dashboard/dashboard-assets.ts` is auto-generated by `precompile`. Do not edit manually.
- Integration tests require `NVIDIA_NIM_API_KEY` env var.
- `package.json` `bin.claude-nim` points to `./out/cli.js` — ensure `compile` succeeds before publishing.
- `tests/__mocks__/vscode.ts` exports `ExtensionContext` as a class (not plain object) with `subscriptions`, `secrets`, `globalState`, `workspaceState`, `storagePath`, `asAbsolutePath`.
- `jest.config.js` sets `testTimeout: 30000` and `restoreMocks: true` — adjust if tests time out on slow machines.
- `eslint.config.js` uses `tsPlugin.config()` syntax compatible with ESLint v10 + typescript-eslint v8.
